CONTINUATION PROMPT — Health Mode Architecture

This section defines the structure, navigation logic and content architecture of the Health Mode of the ablute_ website.

Health Mode represents the medical and scientific dimension of the platform, focusing on preventive health monitoring, biosensing technologies and clinical applications.

The visual design of Health Mode must follow a clean medical interface style, similar to digital health platforms and medical device companies.

Key design characteristics:

• light backgrounds
• clinical blue and teal palette
• soft gradients
• medical dashboard visual elements
• structured information cards

The layout style should resemble the attached interface examples, including biomarker dashboards, health score cards and medical UI blocks.

MAIN HEALTH PAGE

The Health Mode main page introduces the concept of transforming bathrooms into health interfaces.

Headline concept:

Transforming bathrooms into health interfaces

Subheadline:

The ablute_ platform integrates advanced sensing technology to enable continuous, non-invasive health monitoring.

Visual element:

A health dashboard style mobile interface showing biomarker insights.

The mobile interface must be interactive.

When clicked:

It opens a page inviting users to download the mobile application.

If the application is not yet available, display a message:

"The application is not yet available. Leave your email to be notified when it becomes available."

Include a simple form:

Name
Email

FLOATING TECHNOLOGY ELEMENTS

Floating UI elements appear near the mobile interface.

Example:

Lab-grade Technology

When clicking:

AI → opens the section Product / AI

Lab-grade Technology → opens Product / Biosensing

These floating UI elements visually reinforce the platform architecture.
