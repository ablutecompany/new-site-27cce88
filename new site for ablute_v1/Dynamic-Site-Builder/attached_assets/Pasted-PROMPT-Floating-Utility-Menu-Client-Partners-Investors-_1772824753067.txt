PROMPT — Floating Utility Menu (Client / Partners / Investors / Contacts)

Use Anti-Break Editing Rules.

Modify only the utility access system of the website by implementing a floating utility menu button.

Do NOT modify:

• homepage hero section
• typography system
• product animations
• main navigation (Product / Company / Projects)
• Bathroom Mode / Health Mode switch
• layout of existing sections.

OBJECTIVE

Create a floating utility menu system that provides access to:

• client area
• partners & suppliers
• investors
• contacts.

This menu must function as a secondary access interface, independent from the main navigation.

FLOATING MENU BUTTON

The menu must appear as a floating fixed button.

Position

bottom-right corner of the screen

The button must remain visible during:

• scroll down
• scroll up
• page transitions
• section animations.

VISUAL STYLE

The floating button must be highly visible regardless of background.

Requirements:

• circular or rounded button
• premium minimal design
• strong contrast
• subtle shadow or glow
• modern technology aesthetic.

The button must remain visible over:

• dark sections
• light sections
• image backgrounds
• animated sections.

DEFAULT STATE

Default state shows only the icon.

The icon should resemble a minimal menu / utility icon.

Example:

three horizontal lines
or modern minimal grid icon.

INTERACTION BEHAVIOUR

The button works as a toggle.

Click once:

open menu panel.

Click again:

close menu panel.

When closed:

only the floating icon remains visible.

PANEL ANIMATION

When opened, the panel must appear as a roll-up utility panel.

Animation style:

• smooth expansion from the floating button
• modern UI motion
• fast but elegant transition.

The panel must feel like a utility drawer, not a full-screen menu.
